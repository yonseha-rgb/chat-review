## 1. 왜 채팅 후기 고정을 고민하게 되었는가?

> 💡 ASIS 좋은 후기가 채팅에 한 줄 뜨고 2초 만에 사라지지만, TOBE 채팅 후기 고정은 후기를 '흘려보내는 것'에서 '보여주는 것'으로 전환하는 전략

### 방송 중 가장 강력한 구매 설득은 '다른 유저의 후기'인데, 지금은 그냥 흘러가고 있다.

- 방송 시작 전에는 카운트다운, 채팅, 사전 질문을 통해 기대감과 상호작용을 형성하고
- 방송 시작 시에는 인원이 참여한채로 페이지 전환이나 재진입 없이 라이브 화면으로 자연스럽게 전환되며
- **하지만 방송 중, 유저들이 남기는 가장 좋은 후기는 채팅 속에 묻혀 사라진다.**

구매를 고민하는 유저에게 가장 효과적인 넛지는 셀러의 멘트가 아니라, **다른 구매자의 실제 후기**다.
지금은 이 후기가 채팅 속에서 2~3초 만에 스크롤되어 사라진다.
이걸 붙잡아서 보여주는 것만으로도, 구매 전환에 의미있는 임팩트를 만들 수 있다.

---

### 1-1. 비즈니스 관점: 후기 노출이 곧 매출이다

**"채팅 후기는 소셜 프루프(Social Proof)의 가장 강력한 형태인데, 지금은 낭비되고 있다"**

- **Problem:** 유저가 "이거 진짜 좋아요!", "재구매입니다 ㅋㅋ", "어제 받았는데 대박" 같은 후기를 남겨도, 채팅이 빠르게 흘러가면서 다른 유저들이 볼 수 없음. 특히 동접 200명 이상 방송에서는 채팅 속도가 빨라 후기가 체감 1~2초 만에 사라짐.
- **Solution:** 셀러(매니저)가 좋은 후기를 선택하면, 해당 후기가 일반 채팅보다 눈에 띄게 반복 노출되거나, 고정 컴포넌트로 상시 노출됨.
- **Impact:** 구매를 망설이는 유저에게 실시간 사회적 증거를 제공하여 구매 전환율 상승. 후기가 눈에 보이면 "나도 사볼까?"라는 심리가 작동.

### 1-2. 셀러 관점: "이 후기 좀 고정해주세요"는 이미 있는 니즈다

**셀러/매니저가 방송 중 가장 하고 싶은 것 중 하나가 "좋은 후기 보여주기"다**

- **Problem:** 현재 셀러가 좋은 후기를 활용할 수 있는 방법은, 후기를 읽어주거나 캡처해서 보여주는 것뿐. 방송 흐름이 끊기고, 캡처하는 동안 판매 타이밍을 놓침.
- **Solution:** 매니저가 채팅에서 후기를 탭 한 번으로 고정. 별도의 캡처나 멘트 없이도 후기가 유저에게 자동으로 노출됨.
- **Impact:** 셀러는 판매 흐름을 유지하면서도 후기를 활용할 수 있고, 매니저는 방송 운영의 새로운 무기를 갖게 됨.

> 📌 **클루니(더베이지 인터뷰):** "좋은 후기가 빠르게 지나가 버리는 문제가 있어, 주요 후기를 고정 노출할 수 있는 기능을 라이브스튜디오 내 제공하는 방향 검토"
>
> 📌 **운영팀 피드백:** "방송화면 자체가 복잡해서 실제 효과가 있을지, 더 정신없게 만드는 것 아닌지" → 이 우려를 해소하기 위해 2가지 안을 설계

### 1-3. 유저 관점: "다른 사람도 샀다"는 가장 강한 구매 동기

**유저가 라이브에서 구매를 결정하는 순간은, 셀러 멘트가 아니라 '다른 유저의 반응'을 봤을 때다**

- **Problem:** 채팅이 빠르게 흘러가면, 유저는 "이 상품 괜찮은 건가?"에 대한 사회적 증거를 얻지 못함. 특히 첫 구매 유저일수록 다른 유저의 후기에 의존도가 높음.
- **Solution:** 고정된 후기가 눈에 보이면, 유저는 "아, 다른 사람도 사서 만족했구나"라는 확신을 빠르게 얻을 수 있음.
- **Impact:** 망설이는 시간이 줄고, 구매 결정이 빨라짐. 특히 초반 입장 유저들에게 즉각적인 구매 신호 제공.

## 2. 문제 진단

### a. 좋은 후기가 유실되고 있다
- 채팅 속도가 빠른 방송에서 후기 메시지는 1~3초 만에 화면 밖으로 밀림
- 셀러가 "○○님 후기 감사합니다~" 하고 읽어줘도, 이미 화면에서 사라진 상태
- 후기를 다시 보고 싶은 유저가 스크롤을 올려야 하는데, 이러면 실시간 채팅을 놓침

### b. 셀러의 후기 활용 수단이 없다
- 매니저가 좋은 후기를 발견해도 "보여주는" 도구가 없음
- 현재 가능한 건: 채팅에 텍스트로 다시 써주거나, 셀러가 구두로 읽어주는 것뿐
- 방송 리소스가 후기 전달에 낭비됨

### c. 구매 전환의 골든타임에 사회적 증거가 부재
- 상품 설명이 끝나고 "자, 지금 담아주세요!" 하는 순간이 전환의 골든타임
- 이 순간에 "저 방금 샀어요!", "재구매입니다" 같은 후기가 눈에 보이면 전환율 상승
- 현재는 이 타이밍에 후기가 보이는지 안 보이는지는 순전히 운

### d. 화면 복잡도 우려가 존재한다
- 운영팀 피드백: "방송화면 자체가 복잡해서 실제 효과가 있을지, 더 정신없게 만드는 것 아닌지"
- 이 우려는 타당함 → **2가지 안을 설계하여, A/B로 검증**하는 접근이 필요

## 3. 핵심 가설

> **Core hypothesis**
>
> 셀러(매니저)가 좋은 채팅 후기를 선택하여 고정 노출하면, 방송 중 구매를 망설이는 유저에게 실시간 사회적 증거를 제공하여, **구매 전환율과 장바구니 담기 전환율이 상승**할 것이다. 또한 셀러의 방송 운영 효율이 개선되어 매니저 만족도가 향상될 것이다.

### a. 전환가설 (후기 노출 → 구매 전환)

채팅 후기가 고정 노출된 방송은
후기 고정 없는 방송 대비 **구매 전환율이 n% 상승**할 것이다.

### b. 행동가설 (후기 인터랙션 → 장바구니)

고정된 후기를 탭하여 바텀시트에서 후기를 모아본 유저는
후기를 보지 않은 유저 대비 **장바구니 담기 전환율이 n% 높을** 것이다.

### c. 효율가설 (셀러 운영 효율)

채팅 후기 고정 기능을 사용하는 매니저는
후기를 구두로 읽어주거나 캡처하는 시간이 줄어
**상품 전환 속도가 n% 빨라질** 것이다.

### d. 부작용가설 (화면 복잡도 Guardrail)

후기 고정이 화면을 복잡하게 만들어
**평균 시청 시간이 유의미하게 하락하지 않을** 것이다. (Guardrail)

### ⸻

## 🅰 1안: 채팅 내 반복 노출 (Chat Inline Replay)

### 4-1-A. 컨셉

> **"채팅 흐름 안에서 자연스럽게, 하지만 확실하게 눈에 띄게"**

- 셀러(매니저)가 좋은 후기를 선택하면, 해당 후기가 **일반 채팅 사이에 주기적으로 다시 노출**됨
- 일반 채팅은 회색 닉네임 + 흰색 텍스트이지만, 고정 후기는 **그라데이션 배경 + 핑크 테두리 + 📌 태그**로 확실히 구분됨
- 유저가 이 후기를 탭하면 **바텀시트**가 열리면서 고정된 후기들을 모아볼 수 있음

### 4-2-A. 셀러(매니저) 플로우

```
[방송 중 채팅 모니터링]
    ↓
[좋은 후기 발견] — 매니저가 해당 채팅 탭
    ↓
[다이얼로그 얼럿] — 기존: (1)5분 음소거 (2)방송끝까지 음소거
                     추가: (3)💬 채팅 후기 고정   ← NEW
    ↓
[(3) 채팅 후기 고정 선택]
    ↓
[고정 완료] — 해당 후기가 설정된 주기로 채팅에 반복 노출 시작
```

### 4-3-A. 유저 플로우

```
[방송 시청 중]
    ↓
[채팅 영역에서 눈에 띄는 후기 발견]
 — 📌 태그 + 핑크/오렌지 그라데이션 배경
 — 닉네임이 오렌지색으로 강조
 — "후기 3건 더보기 ›" 텍스트 노출
    ↓
[후기 탭]
    ↓
[바텀시트 열림] — 타이틀: "채팅 후기"
 — 고정된 후기들이 카드 형태로 나열
 — 프로필 아바타 + 닉네임 + 후기 텍스트 + 시간
    ↓
[바텀시트 닫기] → 다시 방송 시청
```

### 4-4-A. UI/UX 상세

| 항목 | 상세 |
|------|------|
| 노출 위치 | 채팅 영역 내, 일반 채팅 메시지 사이에 끼워넣기 |
| 노출 스타일 | 그라데이션 배경 `rgba(255,45,85,0.18)` → `rgba(255,149,0,0.12)`, 핑크 테두리 1px, border-radius 12px |
| 태그 | 📌 핑크 배지 "후기" 텍스트, 좌측 상단 |
| 닉네임 색상 | `#ff9500` (오렌지) — 일반 채팅 `rgba(255,255,255,0.55)` 대비 확실한 차이 |
| 텍스트 색상 | `rgba(255,255,255,0.95)` — 일반 채팅보다 살짝 밝게 |
| 반복 주기 옵션 | **15초 / 30초(기본) / 45초 / 1분** — 셀러가 설정 가능 |
| 최대 고정 건수 | 동시 최대 **5건** — 주기마다 라운드로빈으로 순환 노출 |
| 탭 인터랙션 | 탭 시 바텀시트 오픈 → 고정된 후기 전체 모아보기 |
| 애니메이션 | `slideUp 0.35s ease-out` — 채팅 올라오는 모션과 동일하되 약간 느리게 |

### 4-5-A. 장점 / 단점

| 장점 | 단점 |
|------|------|
| 채팅 흐름을 깨지 않으면서 자연스러운 노출 | 채팅 속도가 빠르면 여전히 빠르게 밀릴 수 있음 |
| 유저가 "다른 사람도 좋다고 하네" 하고 자연스럽게 인지 | 반복 노출이 스팸처럼 느껴질 수 있음 (주기 조절로 대응) |
| 기존 UI 변경 최소화 — 채팅 안에서만 동작 | 채팅을 안 보는 유저에게는 효과 없음 |
| 추가 컴포넌트 영역 불필요 → 화면 복잡도 증가 없음 | 후기가 채팅 히스토리에 섞여 맥락 파악이 어려울 수 있음 |

## 🅱 2안: 고정 컴포넌트 (Pinned Review Chip)

### 4-1-B. 컨셉

> **"채팅 밖, 상시 보이는 위치에 후기 요약을 한 줄로 고정"**

- 현재 방송 화면 하단에 **스토어 버튼** 위, **Q칩(상품 컴포넌트)** 위에 **새로운 고정 컴포넌트**를 배치
- Q칩과 동일한 사이즈, 인라인 한 줄 구성: **좌측 "채팅후기" 라벨 | 우측 후기 요약 텍스트**
- 유저가 이 컴포넌트를 탭하면 **바텀시트**가 열리면서 후기를 모아볼 수 있음
- 채팅을 안 보는 유저에게도 후기가 상시 노출되는 것이 핵심 차별점

### 4-2-B. 셀러(매니저) 플로우

```
[방송 중 채팅 모니터링]
    ↓
[좋은 후기 발견] — 매니저가 해당 채팅 탭
    ↓
[다이얼로그 얼럿] — (1)5분 음소거 (2)방송끝까지 음소거 (3)💬 채팅 후기 고정
    ↓
[(3) 채팅 후기 고정 선택]
    ↓
[유저 화면 하단에 고정 컴포넌트 노출]
 — "채팅후기 | 이거 진짜 좋아요!! 재구매..."
 — 새 후기 고정 시 텍스트가 슬라이드 전환
```

### 4-3-B. 유저 플로우

```
[방송 시청 중]
    ↓
[화면 하단 — Q칩 위에 고정 컴포넌트 상시 노출]
 — 좌측: "채팅후기" (오렌지 라벨)
 — 우측: 가장 최근 고정된 후기 요약 (한 줄, 말줄임)
    ↓
[고정 컴포넌트 탭]
    ↓
[바텀시트 열림] — 타이틀: "채팅 후기"
 — 고정된 후기들이 카드 형태로 나열
 — 프로필 아바타 + 닉네임 + 후기 텍스트 + 시간
    ↓
[바텀시트 닫기] → 다시 방송 시청
```

### 4-4-B. UI/UX 상세

| 항목 | 상세 |
|------|------|
| 노출 위치 | Q칩 (5만이상 무배 컴포넌트) **바로 위** |
| 컴포넌트 사이즈 | Q칩과 동일 — `padding: 7px 14px`, `border-radius: 20px` |
| 배경 | `rgba(40,40,48,0.7)` + `backdrop-filter: blur(10px)` — Q칩과 동일 |
| 테두리 | `1px solid rgba(255,45,85,0.25)` — 핑크빛 미세한 강조 |
| 좌측 라벨 | "채팅후기" — `#ff9500` (오렌지), `font-size: 11px`, `font-weight: 700` |
| 구분선 | `1px` 세로 디바이더, `rgba(255,255,255,0.12)` |
| 우측 텍스트 | 가장 최근 후기 한 줄 요약, `font-size: 13px`, 말줄임(ellipsis) |
| 화살표 | `›` — `rgba(255,255,255,0.35)` |
| 진입 애니메이션 | `slideIn 0.4s ease-out` — 좌측에서 우측으로 슬라이드 인 |
| 텍스트 전환 | 새 후기 고정 시 페이드 아웃 → 페이드 인 전환 |
| 탭 인터랙션 | 탭 시 바텀시트 오픈 → 1안과 동일한 후기 모아보기 |
| 후기 없을 때 | 컴포넌트 자체가 미노출 (hide) |

### 4-5-B. 화면 레이아웃 (위→아래 순서)

```
┌─────────────────────────────┐
│  채팅 영역                    │
│  (일반 채팅 메시지들)           │
│                              │
├─────────────────────────────┤  ← 채팅과 하단 영역 사이 마진 충분히 확보
│                              │
│  🆕 [채팅후기 | 이거 진짜...]  │  ← 2안 고정 컴포넌트 (NEW)
│                              │
│  [👩 5만이상 무배  ›]          │  ← Q칩 (기존 상품 컴포넌트)
│                              │
│  [스토어🔴] [채팅입력] [💰] [❤️] │  ← 액션바 (기존)
└─────────────────────────────┘
```

### 4-6-B. 장점 / 단점

| 장점 | 단점 |
|------|------|
| 채팅을 안 보는 유저에게도 상시 노출 | 화면에 컴포넌트 하나가 추가됨 → 복잡도 증가 우려 |
| 한 줄 요약으로 깔끔 — 관심 있으면 탭해서 상세 확인 | 고정 후기가 없을 때 빈 공간 발생 (hide로 대응) |
| 상품 컴포넌트(Q칩) 바로 위 → 구매 동선과 가까움 | 상품 정보 영역을 가릴 수 있음 |
| 셀러가 후기를 상시 무기로 사용 가능 | 후기 텍스트가 짧은 경우 임팩트 약할 수 있음 |

## 5. 1안 vs 2안 비교 및 제안

| 비교 항목 | 1안 (채팅 내 반복) | 2안 (고정 컴포넌트) |
|---------|-----------------|------------------|
| 노출 방식 | 채팅 안에서 주기적 반복 | 채팅 밖 고정 영역에 상시 노출 |
| 노출 보장성 | 중 (채팅 속도에 따라 다름) | **상** (항상 보임) |
| 화면 복잡도 증가 | **낮음** (기존 채팅 안) | 중 (컴포넌트 1개 추가) |
| 채팅 비시청 유저 도달 | ✗ 불가 | **✓ 가능** |
| 구매 동선 근접도 | 중 (채팅 영역) | **상** (스토어 버튼 바로 위) |
| 자연스러움 | **상** (채팅 흐름 유지) | 중 (별도 UI 인지 필요) |
| 구현 난이도 | 중 (타이머 기반 채팅 삽입) | 하 (컴포넌트 추가) |
| 바텀시트 | ✓ 동일 | ✓ 동일 |

### 💡 제안: 1+2안 동시 적용 (프로토타입 3번 탭) → A/B/C 테스트

| 그룹 | 적용 | 목적 |
|------|------|------|
| A (Control) | 후기 고정 없음 | 기존 대비 효과 측정 |
| B | 1안만 적용 | 채팅 내 반복의 단독 효과 |
| C | 2안만 적용 | 고정 컴포넌트의 단독 효과 |
| D | 1+2안 동시 | 시너지 효과 및 화면 복잡도 허용 범위 검증 |

## 6. 공통: 바텀시트 (후기 모아보기)

1안이든 2안이든, 유저가 후기를 탭하면 **동일한 바텀시트**가 열린다.

│  │ 🟣 달콤한인어공주            │  ← 아바타 + 닉네임

### 6-2. 바텀시트 스펙

## 7. Acceptance Criteria (AC) — MVP

### 7-1. 셀러(매니저) Side

| AC | 상세 |
|----|------|
| AC-S01 | 매니저가 채팅에서 유저를 탭하면 기존 다이얼로그에 "💬 채팅 후기 고정" 항목이 추가되어 있다 |
| AC-S02 | "채팅 후기 고정"을 선택하면 해당 채팅이 고정 후기로 등록된다 |
| AC-S03 | 동시 최대 5건까지 고정 가능하다. 6건째 시도 시 "최대 5건까지 가능합니다. 기존 후기를 해제해주세요." 안내 |
| AC-S04 | 고정된 후기는 매니저 화면에서 리스트로 확인 가능하며, 개별 해제 가능하다 |

| AC-S06 | 방송 종료 시 모든 고정 후기가 자동 해제된다 |

| AC | 상세 |

## 9. Edge Case

| 케이스 | 처리 방안 |
|--------|-----------|
| 고정 후기 유저가 방송 중 탈퇴 | 닉네임 "탈퇴한 회원"으로 표시, 후기는 유지 |
| 고정 후기 유저가 채팅 차단당함 | 고정 후기 자동 해제, 매니저에게 안내 |
| 동시 5건 초과 고정 시도 | "최대 5건까지 가능합니다." 안내, 기존 후기 해제 유도 |
| 방송 종료 | 모든 고정 후기 자동 해제 |
| 고정 후기 내용이 부적절한 경우 | 매니저가 수동 해제 (실시간 판단) |
| 2안 컴포넌트 + Q칩 동시 노출 시 영역 겹침 | Q칩 위 마진 확보 (최소 8px), 고정 컴포넌트와 채팅 영역 마진 충분히 확보 |
| 네트워크 지연 | 고정 후기 캐시 유지, 재연결 시 동기화 |
| 1안 반복 노출 중 방송 종료 | 즉시 반복 중단, 마지막 노출 건 유지 후 소멸 |

## 10. 위험요인 & 대응책

- **화면 복잡도 증가:** 운영팀 우려 사항 → A/B 테스트로 시청 시간 하락 여부 검증. Guardrail 지표로 관리
- **후기 조작 (셀러 자작극):** 셀러가 지인을 동원해 가짜 후기 → MVP에서는 매니저 판단에 위임. 추후 구매 인증 후기만 고정 가능하도록 업그레이드 검토
- **반복 노출 스팸감 (1안):** 30초 기본 주기가 짧다고 느낄 수 있음 → 셀러별 주기 조절 가능, 유저 반응(이탈률) 기반 자동 조절 검토
- **구매 전환 기여 측정 어려움:** 후기 고정과 구매 사이 직접적 인과관계 입증 어려움 → 후기 바텀시트 오픈 → 장바구니 담기 경로 추적으로 간접 측정
